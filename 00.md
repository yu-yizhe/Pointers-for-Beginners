# 前言

> 《语言学纲要》给出了“语言”的官方定义：语言是人类社会约定俗成的，音义结合的全民性符号系统，是人类最重要的交际工具、思维工具。

> 无论是 C 语言，还是人们耳熟能详的 Python 语言，共属于一个概念叫“编程语言”。

> Claude Code、Codex、Cursor 风靡全球，真正实现了人们此前“一句话一个前端网页”的调侃。

早在很久以前，就有先知预测 AI 会重构人类社会。可“ AI 时代”真的到来了，却又是另一番不真实感。如果依旧觉得 AI 会和 2000 年的互联网一样是场泡沫，那请先以身入局，再下结论。

对绝大多数人而言，他们最关心的可能是 AI 是否会像互联网上不断翻炒的一样，会导致自己失业。到了今天这个时间点，我想很多人心中都有了答案。然而，出乎很多人意料的是，AI 最先替代的竟然会是视觉设计、程序员......但也是在意料之中的。

原因有以下三点：
> 01.工作相关数据开放度高
> 
> 02.工作结构化程度高
> 
> 03.结果容易量化

当然，并不是说 AI 会替代一个岗位中的所有人，至少目前在程序员这个行业当中，以淘汰低端程序员为主。但 AI 在 coding 这一块的发展速度以及潜力是有目共睹的，这也是为什么当 “vibe-coding” 一词被 Andrej Karpathy 提出后，越来越多编程小白也可以搓出自己的产品。

> Andrej Karpathy（2025-02-02 X 发文）：
> 
> _“There is a new kind of coding I call ‘vibe coding’, where you fully give in to the vibes, embrace exponentials, and forget that the code even exists.”_
> 
> “如今出现了一种全新的编程模式，我将其称作氛围感编程。全然沉浸创作状态，顺应指数级迭代逻辑，不必纠结代码本身。”

可随着越来越多人在互联网上鼓吹，“程序员的好日子到头了...今天我仿着 Apple.com vibe-coding 了一个网站...以后谁还学编程啊...”。我下意识感觉有点不对劲。真的不需要学编程了吗？思来想去之后，我给出一个我的答案：
> 不仅要学，比以前学的还要更深入

人们普遍的共识是：一个事物越稀缺，那么多数情况下就越有价值。

编程能力是互联网时代最重要的能力，甚至不需要加“之一”，难道其价值真的就一瞬间跌入谷底了？仔细思考就会发现 99% 的情况之下，随波逐流一定是错的（尽管多数人不愿承认）。举个简单的例子就是，一个散户，一边和多数散户一样去追涨杀跌，一边又幻想自己赚得盆满钵满，那最后能输得不只剩条裤衩子就不错了，够贴切了吧。

所以，如果人人都能够靠一句话就做出一个脚本，打造一个网页，甚至多说几嘴就搓出一个APP，那又有什么稀缺的呢？你可能在互联网上看到说：未来缺的不是解决问题的人，而是提出问题的人。但我想，一个想法，在如此透明的互联网上，你那怕憋着，憋坏了，说不定明天就有人 vibe-coding 出来并顺手开源了。这个世界并不缺那些天马行空的想法，毕竟不论你我，闲下来的时候也没少做白日梦吧。

再者，提到程序员，往往给人一种很有智慧感觉，这倒不全是刻板印象，因为掌握编程语言和没掌握编程语言确实是两个不同的世界。尴尬的是，这种微妙的区别只可意会，不可言传。而意会的唯一前提便是已经掌握了至少一门编程语言。因此，如果你目前确实是一名编程语言小白，我在这里也只能粗浅地点到为止，你想啊，可以多出一种看待问题的视角，何况还是个极具生产力的视角。（如果这还是没有点燃你心中那团火，不足以让你克服心中的恐惧，是的，就是恐惧，请勇敢承认，这很重要。推荐你可以从一门简单且非常实用的 Markdown 标记语言开始）。程序员的智力水平怎么说也不会太低，既然我们都能用明白，那程序员耍得不更溜？何况互联网行业对生产力工具的推行往往具有领先性，可能我们使用的都是他们玩剩下的。
> 人最可怕之处就是有傲慢之心，更可怕的是还不自知。
> 
> 请永远保持一颗学习之心，AI时代，怎么学都不够，难道不是吗？

既如此，哪怕你今天用 Cursor 写了个自动化脚本，明天又和 Claude Code 开发了一个微信小程序，其实也没什么好过分吹嘘的，这都应该感谢时代，庆幸自己身处这样一个大好的时代，而非成天抱怨环境有多差。因为，以前一个纯代码小白想要搭建一个自动化工作流，或许得先学习半年到一年 Python语言 ，然后在这个过程中脑海中的想法、完美的执行方案早就烟消云散了。而现在仅需要一两天的学习投入，一个此前仅仅是想想，但绝无可能实现的事情就那么被实现了。

有人可能要说了，我知道确实没有什么好过分吹嘘的，但还是想要发出来，刷波存在感。相信我，在这个时代，这不仅没有任何问题，反倒是明智的选择。当然，这份优越感实际上是建立在信息差的基础之上的。即你比别人先去了解了这个 AI工具，并使用了。然而随着时间的推移，大家最后都会用上 AI 的，这是必然的趋势。到那个时候你的优越感也便不复存在了。

那怎么办？可不就是很多 “傻子” 因为自己不敢学而劝大家都别学的编程吗？所以利用好 AI ，以及这段时间差，深度掌握几门计算机语言（只要掌握了一门，后面学起来就是一通百通），把壁垒建立起来，赚到时间，然后接着学接着赚呗。
