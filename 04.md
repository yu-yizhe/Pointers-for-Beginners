
#### How

- 赋值
    

> ```C
> //一般正常赋值
> int* p = &a;
> 
> 需要注意的是'&'（取地址符号）的使用前提：不能给临时的、非左值（没有内存地址的数值）强行取地址。
> 
> 看到临时的、非左值（没有内存地址的数值），很多人第一反应就是“野指针”，然而这是错误的。
> 野指针本身是一个合法的指针变量，它有内存地址，但指向的地方是无效的。因此，是可以取野指针的地址的。
> 
> ------------------------------------------------------------------------------------
> //指针与指针之间的赋值
> 把一个指针变量中存储的「内存地址值」，拷贝给另一个同类型指针变量。
> 即：只拷贝地址编号，不拷贝指向的数据本身。
> 
> 01.同类型指针之间赋值
>     int a = 100;
>     int *p1 = &a;  
>     int *p2;      // 定义两个同类型整型指针
>     p2 = p1;      //语法完全合法，拷贝地址值，共享同一块内存
>     
> 理论上讲，野指针之间可以赋值，只是共享非法地址，不能解引用，且会警告。
> 
> 02.不同类型指针之间赋值
>     int  *p1;
>     char *p2;
>     p2 = p1;   // 跨类型指针赋值，属于隐式转换，不规范、有警告
> 
> 03.万能指针：(void*)
> 可以存放任意类型变量的地址。
> 任意类型指针 都可以直接赋值给(void*)且无警告、不用强转。
> 万能指针赋值给其他指针，需要强制转换。
> 无论赋值与否，void* 不能直接解引用 *p因为编译器不知道它指向的数据占几个字节、是什么类型。
> 同样的，void* 不能做指针算术运算（p++/p-- 报错）没有类型宽度，不知道一步跳几字节。
> #include <stdio.h>
> int main()
> {
>     int a = 10;
>     void *p;             // 万能指针
>     p = &a;              // int* 赋给 void* 
>     int *q = (int *)p;   // 要强制转回原类型才能用
>     return 0;
> }
> 
> 最后请注意“常量指针”与“指针常量”之间的区别
> 常量指针：指向常量的指针(const int *)。属于同类型指针之间的赋值。
> 指针常量：int *const
>     int a = 10, b = 20;
>     int *const p1 = &a;
>     int *const p2 = &b;
>     p2 = p1;           
>     //报错：指针常量本身不可修改，不能赋值
> ```

- 有关加减问题
    

> ```C
> 01.指针相减
> 指针相减，即：计算两个地址之间相差的元素个数。
> 但依旧有两个前提条件：
> ｜必须是同类型指针
> ｜必须指向同一块连续内存（同一个数组）
> 结果 = 地址差值 / 指针指向类型的大小
> 
> 注：两个地址编号相加，没有任何意义。因此，C语言标准明确禁止“指针 + 指针”运算。
> 
> 02.“p[下标]”写法：
> p[i] 等价于 *(p + i)，且只要是指针类型，都支持 p[i] 下标写法。
> 下标本质就是指针偏移 + 解引用。
> 
> p[i]  =  *(p + i)
> 步骤：
> ｜先算 p + i：指针向后跳 i 个元素
> （指针运算规则：p + i 不是地址直接加 i 个字节，而是：地址 + i × 指向类型大小）
> ｜再 * 解引用：取出这个地址里的数值
> 如：
> int *p;
> p 是int 类型指针，一次跳跃 4 字节（一个 int 大小）
> 因此，p 是地址，p[i] 是地址对应的值。
> ```

- 指针 + 数组
    

> ```C
> 一、
> 我们此前讲到过：C语言中，数组名在多数情况下会自动降级为指向数组首元素的指针。
> 那么数组名不会自动降级为指向数组首元素指针的少数情况是什么呢？
> 01.sizeof(数组名)
> 计算整个数组占用的总字节数
> #include <stdio.h>
> int main() {
>     int arr[5] = {1,2,3,4,5}; 
>     printf("sizeof(arr) = %zu\n", sizeof(arr));  
>     return 0;
> }
> //输出：20（int=4字节，5*4=20，为整个数组的大小）
> ---
> 02.&数组名
> 取的是整个数组的地址（类型是数组指针），而不是首元素的地址。
> #include <stdio.h>
> int main() {
>     int arr[5] = {1,2,3,4,5};
>     printf("&arr      = %p\n", &arr);      // 数组整体的地址
>     printf("arr       = %p\n", arr);       // 首元素地址（降级后）
>     printf("&arr[0]   = %p\n", &arr[0]);   // 首元素地址    
>     printf("&arr + 1  = %p\n", &arr + 1);  // 跨度：整个数组（+20字节）
>     printf("arr + 1   = %p\n", arr + 1);   // 跨度：一个元素（+4字节）
>     return 0;
> }
> //输出：
> &arr      = 0061FF10  // 数组起始地址（数值上和首元素地址一样，但含义、类型完全不同）
> arr       = 0061FF10  // 首元素地址
> &arr[0]   = 0061FF10  // 首元素地址
> &arr + 1  = 0061FF24  // 0061FF10 + 20字节（跳过整个数组）
> arr + 1   = 0061FF14  // 0061FF10 + 4字节（跳过1个int元素）
> 
> 注：0061FF10、0061FF24 是内存地址，每次运行、每台电脑都会不一样，这是正常的。
> 
> ------------------------------------------------------------------------------------
> 二、
> 在上面的内容中，我们前向引用了“数组指针”这个概念，从排列组合的角度来讲，“指针”、“数组”两个词语应当有两种组合方式，即：指针数组、数组指针。这是两个极为重要但又极易混淆的概念。
> 
> ｜指针数组
> 即：存着指针的数组
> 举例：
> int *p[5] 
> 变量名p先和右边的[5]结合为p[5]，说明p是一个包含 5 个元素的数组。
> 再和左边的*结合为*p[5]，说明数组里的每个元素，都是int类型的指针。
> 最终：p是一个由「指向 int 类型的指针」组成的数组。
> 
> #include <stdio.h>
> int main() {
>     int a = 10, b = 20, c = 30;
>     int *p[3] = {&a, &b, &c};       // 指针数组：数组的每个元素，都是int*类型的指针 
>                                     // 验证：p是数组，sizeof(p) = 3 * sizeof(int*)
>     printf("sizeof(p) = %zu\n", sizeof(p));           // 64位系统输出24（3*8）
>     printf("p[0] = %p, *p[0] = %d\n", p[0], *p[0]);   // 输出a的地址、10
>     return 0;
> }
> //输出：
> sizeof(p) = 24
> p[0] = 0x7fffa30c17c4, *p[0] = 10
> 
> ---
> ｜数组指针
> 即：指向数组的指针
> 举例：
> int (*p)[5]
> 变量名p先和括号里的*结合为(*p)，说明p是一个指针。
> 再和右边的[5]结合为(*p)[5]，说明这个指针指向的，是一个包含 5 个 int 元素的数组。
> 最终：p是一个「指向包含 5 个 int 元素的数组」的指针。
> 
> #include <stdio.h>
> int main() {
>     int arr[5] = {1,2,3,4,5};
>     int (*p)[5] = &arr;       //数组指针：指向一个包含5个int元素的数组,&arr是整个数组的地址
>                               // 验证：p是指针，sizeof(p) = 指针大小（64位=8字节）
>     printf("sizeof(p) = %zu\n", sizeof(p));   // 输出8
>     printf("p = %p, p+1 = %p\n", p, p+1);     // p指向类型是整个数组，所以跳过整个数组
>     printf("(*p)[0] = %d\n", (*p)[0]);        // *p 就是数组本身,输出1
>     return 0;
> }
> //输出：
> sizeof(p) = 8
> p = 0x7fff52f94fe0, p+1 = 0x7fff52f94ff4
> (*p)[0] = 1
> 
> ------------------------------------------------------------------------------------
> 三、
> 上述讲解当中我们都是拿一位数组来举例，没有提到“二/多维数组”，核心原因是在 C语言中，会将二维数组当作一位来访问。即：按行优先存储。
> #include <stdio.h>
> int main() {
>     int a[2][3] = {{1,2,3}, {4,5,6}};
>     // 验证行优先存储：连续内存
>     printf("a[0][0] = %p, a[0][1] = %p, a[0][2] = %p\n", &a[0][0], &a[0][1], &a[0][2]);
>     printf("a[1][0] = %p, a[1][1] = %p, a[1][2] = %p\n", &a[1][0], &a[1][1], &a[1][2]);
>     // 验证下标与指针的等价性
>     printf("a[1][2] = %d\n", a[1][2]);
>     printf("*(*(a + 1) + 2) = %d\n", *(*(a + 1) + 2));
>     printf("*(&a[0][0] + 3*1 + 2) = %d\n", *(&a[0][0] + 3*1 + 2));
>     return 0;
> }
> //输出：
> a[0][0] = 0x7ffdabcdef00, a[0][1] = 0x7ffdabcdef04, a[0][2] = 0x7ffdabcdef08
> a[1][0] = 0x7ffdabcdef0c, a[1][1] = 0x7ffdabcdef10, a[1][2] = 0x7ffdabcdef14
> a[1][2] = 6
> *(*(a + 1) + 2) = 6
> *(&a[0][0] + 3*1 + 2) = 6
> 
> 即：a[i][j] <=> *(pointer + 3*i + j) 「此3来源于数组的列数」
> 
> ------------------------------------------------------------------------------------
> 四、
> 关于static 修饰局部数组，解决数组跨域问题
> 如果你在函数里定义了一个局部数组，然后把它的指针返回给函数外部使用。由于函数执行结束后，局部数组被销毁，指针变成野指针，后续访问会直接崩溃。
> 本质上是：数组的生命周期，和指针的使用场景跨域
> 举例：
> #include <stdio.h>
> int* get_array() {
>     int arr[3] = {1,2,3};   // 局部数组：存储在栈区，函数结束后自动销毁
>     return arr;             // 返回数组首元素指针
> }
> int main() {
>     int *p = get_array();   // 这里访问p指向的内存：arr已经被销毁，p是野指针！
>     printf("%d\n", p[0]);   // 输出乱码、直接崩溃，这就是「跨域问题」
>     return 0;
> }
> 
> 为什么会跨域？
> 局部数组arr的生命周期：仅在get_array函数执行期间有效，函数return后，栈空间被释放，arr就不存在了。
> 指针p在main函数里使用时，指向的是一块已经被释放的无效内存，数组的生命周期，跨出了它的作用域。
> ---
> 而使用static修饰后：
> #include <stdio.h>
> int* get_array() {
>     static int arr[3] = {1,2,3};       // static数组：存储在静态区，函数结束后不会被销毁
>     return arr;                        
> }
> int main() {
>     int *p = get_array();
>     printf("%d\n", p[0]);              // 正常输出1，不会崩溃
>     return 0;
> }
> ---
> 当然如果你不想用static修饰，最安全的替代方案是：不要让函数返回局部数组的指针，而是把数组定义在调用它的上层函数里（比如main函数）或是直接定义为全局数组，再把数组作为参数传给函数使用。
> #include <stdio.h>
> void process_array(int arr[], int len) {    // 仅在函数内使用传入的数组，不返回局部数组指针
>     for (int i = 0; i < len; i++) {
>         printf("arr[%d] = %d\n", i, arr[i]);
>     }
> }
> int main(void) {
>     int arr[3] = {1, 2, 3};      // 数组定义在main中，生命周期随main函数，调用时传入函数
>     process_array(arr, sizeof(arr)/sizeof(arr[0]));
>     return 0;
> }
> 
> #include <stdio.h>
> int arr[3] = {1, 2, 3};   // 全局数组存储在静态存储区，生命周期贯穿整个程序
> int* get_array(void) {    // 直接返回全局数组的指针，完全安全
>     return arr;
> }
> int main(void) {
>     int *p = get_array();
>     printf("p[0] = %d\n", p[0]);   // 正常输出1
>     return 0;
> }
> ```

- 指针 + 函数
    

> ```C
> 一、
> 与“指针 + 数组”类似的，“指针”与“函数”的组合也有两个概念：
> ｜指针函数
> 即：返回值为指针类型的函数
> 示例：
> int *func(int a, int b)
> 变量名func先和右边的(int a, int b)结合为func(参数列表)，说明func是一个函数。
> 再和左边的*结合为*func(...)，说明这个函数的返回值是一个指针。
> 最终：func是一个「返回值为指向 int 类型的指针」的函数。
> 
>  #include <stdio.h>
> int* get_max(int *a, int *b) {      // 指针函数：本质是函数，返回值是int*类型的指针
>     if (*a > *b) {
>         return a; 
>     } else {
>         return b;
>     }
> }
> int main() {
>     int x = 10, y = 20;
>     int *p = get_max(&x, &y);       // 接收指针函数的返回值（指针）
>     printf("*p = %d\n", *p);        // 输出20
>     return 0;
> }
> //输出：
> *p = 20
> ---
> ｜函数指针
> 即：指向函数的指针
> 示例：
> int (*func)(int a, int b)
> 括号改变优先级，变量名func先和括号里的*结合为(*func)，说明func是一个指针。
> 再和右边的(int a, int b)结合为(*func)(参数列表)，说明这个指针指向的是一个函数，这个函数的参数是 2 个 int，返回值是 int。
> 最终：func是一个「指向参数为 int、返回值为 int 的函数」的指针。
> 
> #include <stdio.h>
> int add(int a, int b) {   // 待指向的目标函数
>     return a + b;
> }
> int main() {
>     int (*func_ptr)(int, int) = &add; 
>     // 函数指针：本质是指针，指向一个「参数2个int、返回值int」的函数，&add是函数的地址
>     // 验证：func_ptr是指针，sizeof(func_ptr) = 指针大小（64位=8字节）
>     printf("sizeof(func_ptr) = %zu\n", sizeof(func_ptr));   // 输出8
>     int result = (*func_ptr)(2, 3);                         // 通过函数指针调用函数
>     printf("result = %d\n", result);                        // 输出5
>     return 0;
> }
> //输出：
> sizeof(func_ptr) = 8
> result = 5
> 
> ------------------------------------------------------------------------------------
> ```

回顾“指针数组”、“数组指针”、“指针函数”、“函数指针”这四个易混淆概念，我们不难发现"()"的作用功不可没。

在 C 语言中，[]（数组下标）和()（函数调用）的优先级，高于*（指针解引用）。

而括号()可以改变优先级，使得括号内的内容优先结合。
