- 何时要用到指针？
    

这个问题和上一个问题实际上是顺承关系，当你搞明白为什么要有“指针”之后，什么时候会用到指针就更易理解了。

> ｜01.跨越函数管辖范围修改变量
> 
> ```C
> 举例：「交换两个数」
> 
> 代码段一:
> #include <stdio.h>
> void swap_by_value(int x, int y) {
>     int temp = x;
>     x = y;
>     y = temp;
> }
> int main() {
>     int a = 1, b = 2;
>     swap_by_value(a, b); 
>     printf("a=%d, b=%d\n", a, b);
>     return 0;
> }
> //输出结果：a=1, b=2
> 是的你没有看错。
> 
> 在解释代码段一之前，我们先同步一个前置补丁：
> 任何函数都有其自己的管辖范围。
> 比如我们最常用到的 main() 函数。其管辖范围就是花括号中的所有内容。
> （通过点击小三角你可以将其管辖的代码收起来）
> 
> #include <stdio.h>
> int main(){
> 
> }
> 
> 代码段一解释：
> 自定义函数中 参数 x 和 y 是局部变量，只会在 swap_by_value 函数的栈内存里创建。
> 调用函数时，只是把 a=1、b=2 的数值拷贝给了 x 和 y。
> 函数里交换的只是 x 和 y 这两个副本，完全不影响外面的原变量 a 和 b。
> 函数执行结束，x 和 y 会被自动销毁，外面的 a 和 b 从头到尾都没变化。
> 
> ---------------------------------------------------------------------------------
> 代码段二:
> #include <stdio.h>
> void swap_by_pointer(int *x, int *y) {
>     int temp = *x; 
>     *x = *y;       
>     *y = temp;    
> }
> int main() {
>     int a = 1, b = 2;
>     swap_by_pointer(&a, &b); 
>     printf("a=%d, b=%d\n", a, b);
>     return 0;
> }
> //输出结果：a=2, b=1
> 
> 代码段二解释：
> 自定义函数中 参数 x 和 y 是指针变量，接收的是原变量 a 和 b 的内存地址。
> 调用函数时，传的不是数值，而是 a 和 b 在内存里的地址。
> 函数里通过(*x)、(*y)解引用，直接通过地址找到外面的原变量 a 和 b，直接修改它们的值。
> 函数执行结束，a 和 b 的值已经被永久修改，所以输出结果是交换后的数值。
> 
> ---------------------------------------------------------------------------------
> 或许添上这段代码可以彻底解除你心中的疑虑。
> 
> 代码段三:
> #include <stdio.h>
> int main() {
>     int a = 1, b = 2;
>     int temp;   
>     temp = a;
>     a = b;
>     b = temp;
>     printf("a=%d, b=%d\n", a, b);
>     return 0;
> }
> //输出结果：a=2, b=1
> 即：直接在main函数内部交换
> 
> ---------------------------------------------------------------------------------
> 总结：普通变量传参是拷贝一份值，函数里改的只是副本，外面原值不变。但有了指针，传地址，函数可直接操作原变量本身。
> ```

> ｜02.减少内存消耗
> 
> 当一个结构体或数组特别大时，直接传参会复制一大块内存，既慢也费空间。选择传指针，可实现只传一个地址(4或8个字节)，极省内存、速度快。
> 
> 如：数组传参时退化为数组首元素的指针。
> 
> ```C
> #include <stdio.h>
> void print_arr(int arr[], int len) {

>         printf("%d ", arr[i]);
>     }
> }
> int main() {
>     int nums[3] = {1, 2, 3};

> 举个具体的例字往往比说多少话都来的靠谱：
> //场景1：字符串常量 "1345" 本身是什么？
> "1345" 是字符串常量，编译器会自动在内存里生成一块只读区域，存5个字节：'1' '3' '4' '5' '\0'。
> 它的本质：匿名字符数组（编译器偷偷帮你创建的、没有名字的字符数组）
> 它的地址：永远是第一个字符 '1' 的内存地址
> 它的类型：在表达式里，会立即自动降级为指向首字符的指针，也就是 char*，指向 '1'。
> 
> //场景2：char arr[] = "1345"; 里的 "1345" 是什么？
> 这里的 "1345" 是用字符串常量初始化字符数组：
> 编译器先创建匿名常量数组 {'1','3','4','5','\0'}
> 把这5个字节拷贝到你定义的 arr 数组的内存里
> arr 是有名字符数组，类型 char[5]，可读写
> 后续 arr 在表达式里，自动降级为指向首元素 '1' 的指针。
> 
> //场景3：char *p = "1345"; 里的 "1345" 是什么？
> 这里的 "1345" 直接降级为指向首字符 '1' 的指针，把这个地址赋值给 p：
> p 是指针变量，存的是匿名常量数组里 '1' 的地址
> 你不能通过 p 修改这个字符串（只读内存），比如 p[0] = '2' 会直接崩溃。   
> ```
> 
> 似乎最后一句话又卡着你了。只读内存？
> 
> 此前，在“如何避免野指针？”这一节当中我们定义了“栈内存”和“堆内存”，可没有提到过只读内存。

事实上，C 语言程序运行时，内存被严格划分为 5 个区域，每个区域有各自所处的位置、读写权限、生命周期、适用场景。

值得注意的是，“只读内存”更确切地说应该是“只读不可写”的内存，而非 C语言中的5个内存区域之一。

如果你将“只读内存”等价为“只读数据段”，这就是混淆了位置和权限这两个概念。

｜位置：所处的内存区域

｜权限：可读写 or 只读

因此正确用法是：先讲位置，再讲权限。

比如：

```C
char *p = "1345";  
p[0] = '2';        
//程序崩溃
//"1345" 是字符串常量，会被编译器存放在程序的只读数据段（RODATA），属于只读内存；p 是一个指针变量，本身存放在栈内存，它存储的是只读数据段中，字符串首字符 '1' 的内存地址。
---
char p[] = "1345"; 
p[0] = '2';       
//正常运行
//"1345" 这个字符串常量依然先被存放在只读数据段，但编译器会自动把这个字符串完整拷贝一份，放到栈内存中创建的数组 p 里；数组 p 本身存放在栈内存，属于可读写内存。
```

- 如何判断什么时候是只读，什么时候是可读写？
    
      01.判断所处内存区域
    
      02.逆向思维 
    
      ｜能改值（比如p[0] = '2'）还不崩溃 -> 可读写
    
      ｜一改就崩溃（段错误）-> 只读内存
    

以后但凡遇到任何有关 C语言内存的问题：

01.看存放位置：它存在程序的哪块内存区域？

02.看读写权限：这个区域是只读，还是可读写？

03.看生命周期：它什么时候申请，什么时候释放？

> 
> ```C++
> C字符串：一串以'\0（空字符）'结尾的字符序列。即：只要满足「字符+\0」，就是C语言意义上的“字符串”。
> 
> 字符数组(char [N]，N是数组总长度）：一块连续的内存空间，存了 字符 + '\0' ，内存可读写，数组名是数组的标识，仅在多数场景自动降级。
> 
> 字符指针(char *）：一个变量，存了字符串首字符的内存地址，本身是指针，指向只读字符串常量，通常不可直接修改。
> 
> 这三者的关系就有点像是：
> 字符串 = 杯子里的水（内容）
> 数组 = 装水的杯子（底层容器）
> 指针 = 指向杯子把手的手指（操作方式）
> 即：水装在杯子里，用手指指着杯子来操作水。
> 
> 最后如果你还是不能理解“数组传参时退化为数组首元素的指针”“字符数组名在表达式里，会自动降级为指向首元素的指针”这两句话，通过以下这段直观的代码或许能帮到你。
> ---
> #include <stdio.h>
> #include <string.h>
> int main() {
>     char arr[] = "1345";
>     char *p = "1345";    
>     printf("&arr[0] = %p\n", &arr[0]);    // 数组首元素'1'的地址
>     printf("arr     = %p\n", arr);        // 数组名降级后的地址
>     printf("p       = %p\n", p);          // 指针p存的地址
>     printf("*arr    = %c\n", *arr);  
>     printf("*p      = %c\n", *p);   
>     return 0;
> }
> //输出：
> &arr[0] = 0061FF1B
> arr     = 0061FF1B
> p       = 00404030
> *arr    = 1
> *p      = 1
> ---
> 解释：
> 0061FF1B、00404030 是内存地址，每次运行、每台电脑都会不一样，这是正常的。
> 
> 在绝大多数表达式场景下，数组名会自动降级为指向数组首元素的指针，也就是 arr 等价于 &arr[0]。
> 因此：&arr[0] 和 arr 的地址永远完全相等。
> 
> char arr[]：是在栈区(stack)开辟了一块独立的数组空间，把字符串 "1345" 拷贝到这块栈内存里，数组是「自己的」，你可以随便改里面的字符（比如 arr[0] = '2'; 完全合法）。
> char *p：是定义了一个指针变量，它直接指向只读常量区(rodata)里的字符串字面量 "1345"，这块内存是只读的，你试图改 p[0] = '2'; 会直接程序崩溃。
> 即：char arr[] = "1345"; 和 char *p = "1345"; 是完全不同的两种语法。
> 因此：p 的地址和 arr 一定不同
> 
> *arr 和 *p 都指向字符串的第一个字符 '1'，因此输出一定都是 1。
> ---------------------------------------------------------------------------------
> 总结：
> 操作数组、字符串的本质就是指针。
> C字符串类型本质是字符数组。
> 数组名本质就是首元素地址遍历数组、遍历字符串，全靠指针偏移实现。
> ```

> ｜04.动态内存分配
> 
> ```C
> 当你需要的内存大小，在程序运行时才知道（比如用户输入数组长度），必须用指针接收动态开辟的内存，而实现这个功能的也就是我们已经熟知的(malloc/free)组合。
> #include <stdlib.h>
> int main() {
>     int len;
>     scanf("%d", &len);
>     int *arr = (int *)malloc(len * sizeof(int));     // 动态开辟int数组，用指针接收首地址
>     for(int i=0; i<len; i++) {
>         arr[i] = i;
>     }
>     free(arr); 
>     return 0;
> }
> 
> (malloc/free)组合全靠指针接收地址。没有指针，就只能用固定大小数组，做不了可变长度、做不了链表、树、数据结构。
> ```
