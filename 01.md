# What

### 什么是指针？

> 指针：一种保存「某变量」地址的变量。
>
> 即：指针是变量。指针保存某变量的地址。
>
> 地址：计算机「给内存中每一个最小存储单元分配的唯一」编号
> 
> ```C
> int* p      //指向「int型变量」的「指针p」     //(int*) = 整型指针类型
> char* c     //指向「char型变量」的「指针c」    //(char*) = 字符型指针类型
> float* f    //指向「float型变量」的「指针f」   //(float*) = 浮点型指针类型
> //即：(类型*) 变量名
> //注：你写的时候不需要有小括号()，我这里加小括号只是为了辅助你理解
> ```

&nbsp;

### 什么是解指针？

> 解「指针」：通过指针保存的地址，取得该地址对应的值。
> 
> ```C
> int x = 1;          //定义int型变量x
> int* p = &x;        //定义指向int型变量x的指针p
> printf("%d", *p);   //输出解指针(*p)的值
> printf("%d", x);    //输出变量x的值
> //正确运行之后两次输出的值相同
> 
> //即：(int*) p = &x; //得到变量x的地址,将这个地址存入指针p
> //  *p == 1 -> True //解「指针p」 <=> 解「变量x的地址」,得到x的值'1'
> ```

&nbsp;

### 什么是二级(多级)指针？

> ```C
> int x = 1;          
> int* p1 = &x;
> int** p2 = &p1;      //指向「指向『int型变量x』的 指针p1」的指针p2
> 
> //即：p1为一级指针，p2为二级指针
> //注：C 语言没有规定指针的最大级数，理论上可以无限嵌套（比如 int***********p），但在实际开发中，超过 3 级的指针几乎不会出现，工程界公认的实用上限是 2 级。
> ```

&nbsp;

### 什么是野指针？

> 野指针：存入了未知的非法内存地址的指针变量。（即：「无指向的」指针）
> 
> - 什么情况会产生野指针？
> 	
>     ```C
>     01.定义指针未进行初始化
>     如：
>     int *p;
>     
>     --
>     02.指向的变量已经销毁
>     如：
>     int *fun()
>     {
>         int a = 10;
>         return &a;  
>     }
>     
>     --
>     03.越界访问、释放后还继续用
>     如：
>     int *p = (int*)malloc(4);
>     free(p);   // free之后p还保留原来地址，但内存已经还给系统，若再使用p就是野指针
>     
>     --
>     ```
>
> - 为何要避免野指针？
>     - 野指针直接 `(*p)` 会报错。
>     - 即：指针必须指向合法的内存地址才能解指针。
>
>  - 如何避免野指针？
>
> 	 ```C
>     请记住以下三条原则：
>     注：原则与 “什么情况会产生野指针？” 中提到的三点一一对应，故而可以对照着来看。
>     --
>     原则一 ｜但凡这个指针现在可能用不到，但在后续可能会用到的，立即将指针赋值为 NULL。
>     如：int* p = NULL; 
>     
>     --
>     原则二 ｜不要返回「局部变量」的地址
>     //如果你不知道“局部变量”请仔细理解下面的解释，若已经掌握请自便。
>     
>     局部变量：函数里面定义的变量。 
>     举例：
>     int *fun()
>     {
>         int a = 10;   // a 是局部变量
>         return &a;  
>     }
>     即：“局部变量”的生命周期只在该函数内部，函数一执行完就立刻从内存消失，地址不能往外传。
>     
>     --
>     原则三 ｜用 malloc 借来内存，free 归还系统后，立即将指针赋值为 NULL。
>     //如果你不知道“'malloc'和'free'这对组合”请仔细理解下面的解释，若以及掌握请自便。
>     
>     在正式解释之前先给总流程：
>     malloc 申请 -> 使用 -> free 归还 -> 指针置 NULL
>     
>     ｜什么是 malloc？
>     malloc：向操作系统临时借来一块「堆内存」。   //不知道什么是“堆内存”？
>     
>     ｜什么是“堆内存”？
>     栈内存、堆内存是 C语言 中的两种内存：
>     - 堆内存（Heap）
> 	  存：malloc 手动申请的内存
> 	  特点：手动申请、手动释放。不会自动回收，不 free 就永远占着。空间极大，适合存大数据、动态数组。链表完全由程序员手动掌控。
>     
>     - 栈内存（Stack）
>     存：局部变量、函数参数
>     特点：自动分配、自动释放。函数一结束，自动回收。空间小，速度快。   //还是理解不了？
>     
>     *类比是一种非常好的辅助理解手段。
>     栈内存 = 酒店临时房间你入住（进函数）自动分配，退房（函数结束）自动清空，不用自己收拾。
> 	  堆内存 = 长期租房自己去找房东租房间（malloc），不住了必须自己退房（free），你不退房房子一直被你占用，别人用不了（内存泄漏）。
>     
>     ｜什么是 free？
>     free：把 malloc 借来的内存，归还给操作系统。
>     因此若只 malloc 不 free，一直借却从不还，最终就会导致程序越跑越卡，内存被占光，即内存泄漏。
>     
>     举例：
>     int *p = (int *)malloc(sizeof(int));   // 1. 借内存
>     *p = 100;                              // 2. 正常使用这块内存
>     free(p);                               // 3. 用完归还
>     p = NULL;                              // 4. 擦除旧地址，防止野指针
>     
>     到这里你可能还是有一点迷糊：free 不是已经将 malloc 借来的内存归还了吗？为啥还要置NULL呢？
>     *请不要因为自己问题多反以为耻，这反映了你真的在思考，另外恭喜你拥有宝贵的「提出问题的能力」！
>     
>     在上述举例完成步骤3后 指针变量p 指向的那块堆内存，已经归还给操作系统，程序不具备访问权限了；指针变量p 中存储的地址数值，依然保持原来的值，不会自动清空；所以如果不置NULL，那么后续再用 (*p) 读写这块内存，就是野指针行为。
>     
>     --
>     ```
